# my-ai-harness

A personal harness for any AI model.

## What
A structured folder that accumulates context, skills, tools, and outputs around one project. The longer you use it, the sharper your AI gets — not because the model changed, but because the harness did.

Currently wired for Claude Code. The structure works with any model.

## How
Five parts, four required and one optional:

- **Knowledge** — what the AI knows about you and your project
- **Skills** — reusable processes for tasks that repeat
- **Tools** — code and integrations the AI can run
- **Outputs** — work product
- **Agents** *(optional)* — orchestrations that combine skills

Nothing is pre-built. You do real work. When something's worth keeping, flag it — the harness captures it as you go.

At session end, ask *"should we update context?"* — anything new gets recorded.

## Why
Most AI usage resets every session. This one compounds. That's the difference.

## Start
Fork. Open Claude Code. Run `/onboard` or just start working.
